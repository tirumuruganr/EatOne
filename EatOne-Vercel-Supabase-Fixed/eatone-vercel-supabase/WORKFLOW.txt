EAT ONE workflow

CUSTOMER SIDE
1. Customer selects products and enters delivery details.
2. Customer clicks "Send Order via WhatsApp".
3. Backend generates an Order ID (EO-ORD-00001...).
4. WhatsApp opens with Order ID + complete order details.
5. No invoice generation is shown to the customer.

ADMIN SIDE
1. Open /admin yourself.
2. Enter the customer's Order ID after verifying payment.
3. Click Confirm Payment & Generate Invoice.
4. Backend generates a separate sequential Invoice ID (EO-INV-00001...).
5. Invoice PDF is generated only from the admin page.

Development:
Terminal 1: node server.js
Terminal 2: npm run dev
Customer: http://localhost:5173
Admin: http://localhost:5173/admin
